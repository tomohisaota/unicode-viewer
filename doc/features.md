# Features

## Unicode 解析

テキストを入力すると、以下の情報をコードポイント単位で表示します。

### グラフェムクラスタの分解

`Intl.Segmenter` を使用して、テキストを「見た目上の 1 文字」（グラフェムクラスタ）に分割します。

```
入力: 👨‍👩‍👧‍👦
結果: 1 グラフェムクラスタ、7 コードポイント
       U+1F468 (👨) + U+200D (ZWJ) + U+1F469 (👩) + U+200D (ZWJ)
       + U+1F467 (👧) + U+200D (ZWJ) + U+1F466 (👦)
```

### コードポイントメタデータ

各コードポイントについて以下の情報を表示します。

| 項目 | 説明 | 例 |
|---|---|---|
| Code Point | Unicode コードポイント | U+0041 |
| Name | Unicode 名 | LATIN CAPITAL LETTER A |
| Category | 一般カテゴリ | Lu (Letter, uppercase) |
| Block | Unicode ブロック | Basic Latin |
| JIS Level | JIS 水準 | 第一水準 |
| UTF-8 | UTF-8 バイト列 | 41 |
| UTF-16 | UTF-16 コード単位 | 0041 |
| レガシーバイト列 | 選択中エンコーディングのバイト列 | 82 60 (Shift_JIS) |

### 特殊文字の表示

- **制御文字** (U+0000〜U+001F): `\u{0000}` 形式で表示
- **スペース** (U+0020): `␣` で可視化
- **全角スペース** (U+3000): `␣` で可視化
- **その他の不可視文字**: Unicode コードポイント表記で表示

## Unicode 正規化

入力テキストに加えて、4 つの正規化形式を並べて表示します。

| 形式 | 名称 | 説明 |
|---|---|---|
| NFC | 正準合成 | 文字を合成形に変換（最も一般的な形式） |
| NFD | 正準分解 | 文字を分解形に変換（基底文字 + 結合文字） |
| NFKC | 互換合成 | 互換文字を正準等価に変換して合成 |
| NFKD | 互換分解 | 互換文字を正準等価に変換して分解 |

### 差分ハイライト

正規化の結果が入力と異なる場合、そのセクション内のセルが amber（黄色系）でハイライトされます。

```
入力: café (NFD の é = e + ◌́)
NFC:  café (4 コードポイント) → é が合成形に → 差分ハイライトなし
NFD:  café (5 コードポイント) → 入力と同じ → 差分ハイライトなし

入力: café (NFC の é = 単一コードポイント)
NFD:  café (5 コードポイント) → e + ◌́ に分解 → 差分ハイライト表示
```

### コピー機能

正規化結果が入力と異なる場合、「入力にコピー」ボタンが表示されます。クリックすると正規化結果を入力欄にコピーし、そのテキストをさらに解析できます。

## レガシーエンコーディング

詳細は [Encodings](./encodings.md) を参照してください。

### エンコーディング選択

ドロップダウンから以下の 8 モードを選択できます。

| モード | 説明 |
|---|---|
| Unicode | レガシーバイト列の代わりに Unicode コードポイントを表示（デフォルト） |
| ASCII | 7-bit ASCII (0x00〜0x7F) |
| Latin-1 | ISO-8859-1 (0x00〜0xFF) |
| Shift_JIS | JIS X 0208 標準 |
| Shift_JIS-2004 | JIS X 0208 + JIS X 0213 第三・第四水準 |
| CP932 | Windows-31J（Microsoft 拡張を含む） |
| EUC-JP | 日本語 EUC |
| ISO-2022-JP | 7-bit の日本語エンコーディング |

### エンコード不可文字の表示

選択中のエンコーディングで表現できない文字には、以下の視覚的フィードバックがあります。

- セル上に赤い `✕` バッジを表示
- セルの枠が赤色に変化
- セクションヘッダーに赤いピルで非対応文字数を表示

### マッピングバリアント

Shift_JIS 系エンコーディング（Shift_JIS, Shift_JIS-2004, EUC-JP, ISO-2022-JP）では、WHATWG (Microsoft) と Unicode.org (JIS 標準) のマッピングを切り替えられます。CP932 では常に Microsoft マッピングが使用されます。

## JIS 水準

日本語の文字に対して、JIS 規格に基づく水準分類と区点情報を表示します。

| 水準 | 規格 | 文字数 | 内容 |
|---|---|---|---|
| 第一水準 | JIS X 0208 | 2,965 字 | 常用漢字を含む頻出文字 |
| 第二水準 | JIS X 0208 | 3,390 字 | やや使用頻度の低い文字 |
| 第三水準 | JIS X 0213 | 1,893 字 | 希少・歴史的な文字 |
| 第四水準 | JIS X 0213 | - | 異体字など |

### 区点 (Kuten)

JIS X 0213 の区点番号を `N面N区N点` の形式で表示します。面 1 は JIS X 0208 互換領域、面 2 は JIS X 0213 追加領域です。

| 文字 | 水準 | 区点 |
|---|---|---|
| 亜 (U+4E9C) | 第一水準 | 1面16区1点 |
| 漢 (U+6F22) | 第一水準 | 1面20区33点 |
| 弌 (U+5F0C) | 第二水準 | 1面48区1点 |
| 𠮟 (U+20B9F) | 第三水準 | 1面47区52点 |
| 丂 (U+4E02) | 第四水準 | 2面… |

マッピングバリアント（WHATWG / Unicode.org）の切替により、Wave Dash 問題の対象文字（7 文字）の区点表示も適切に切り替わります。

## 特殊文字アノテーション

以下の 25 種類の特殊文字に対して、詳細パネル内に注釈を表示します。

### 異体字セレクタ (Variation Selectors)

| 文字 | 範囲 | 説明 |
|---|---|---|
| VS1〜VS14 | U+FE00〜U+FE0D | 標準化された異体字セレクタ |
| VS15 (テキスト) | U+FE0E | 絵文字をテキスト表示に切り替え |
| VS16 (絵文字) | U+FE0F | テキストを絵文字表示に切り替え |
| IVS | U+E0100〜U+E01EF | 漢字の異体字を選択（CJK 統合漢字用） |

### ゼロ幅文字

| 文字 | コードポイント | 説明 |
|---|---|---|
| ZWJ | U+200D | ゼロ幅接合子（絵文字シーケンスなどで使用） |
| ZWNJ | U+200C | ゼロ幅非接合子（合字を防止） |
| ZWSP | U+200B | ゼロ幅スペース（改行可能位置を示す） |
| Word Joiner | U+2060 | 改行禁止のゼロ幅文字 |
| Soft Hyphen | U+00AD | 条件付きハイフネーション位置 |

### Bidi 制御文字

| 文字 | コードポイント | 説明 |
|---|---|---|
| LRM | U+200E | 左から右のマーク |
| RLM | U+200F | 右から左のマーク |
| LRE/RLE/LRO/RLO/PDF | U+202A〜U+202E | 双方向埋め込み・上書き |
| LRI/RLI/FSI/PDI | U+2066〜U+2069 | 双方向分離 |

### その他

- **絵文字修飾子** (U+1F3FB〜U+1F3FF): 肌の色を指定
- **結合キーキャップ** (U+20E3): 前の文字をキーキャップで囲む
- **結合マーク** (U+0300〜U+036F): アクセント記号などの結合文字
- **タグ文字** (U+E0001〜U+E007F): 旗の地域サブ区分などに使用
- **サロゲート** (U+D800〜U+DFFF): UTF-16 のサロゲートペア用コードポイント
- **私用領域** (U+E000〜U+F8FF, U+F0000〜U+10FFFF): ユーザー定義の文字

## 入力機能

### Unicode 表記の自動変換

入力オプションを有効にすると、以下の表記を実際の文字に自動変換します。

| オプション | 入力例 | 変換結果 |
|---|---|---|
| U+XXXX 変換 | `U+1F44D` | 👍 |
| \uXXXX 変換 | `\u0041` | A |

### サンプルメニュー

13 種のプリセットサンプルを用意しています。ロケールに応じて英語/日本語のラベルが表示されます。

サンプルの内容:
- ZWJ シーケンス（家族絵文字など）
- 結合文字（café の NFD 表現）
- 異体字セレクタ（テキスト/絵文字切替）
- Wave Dash 問題（WHATWG vs Unicode.org）
- JIS 水準比較
- 制御文字
- サロゲートペア

### URL パラメータ

入力テキストは URL のクエリパラメータ `?text=...` として保存されます。URL を共有することで、同じテキストの解析結果を他の人と共有できます。

## 国際化 (i18n)

`navigator.language` が `ja` で始まる場合は日本語、それ以外は英語で表示します。

対応範囲:
- UI ラベル（200 以上の翻訳キー）
- ヘルプダイアログ（6 セクション）
- サンプルメニュー（13 項目）
- 特殊文字アノテーション（25 種）

## PWA (Progressive Web App)

- `manifest.json` によるアプリ情報の定義
- Service Worker による基本的なオフライン対応
- ホーム画面追加に対応したアイコン（192x192, 512x512）
- スタンドアロン表示モード

## 共有機能

ヘッダーの共有ボタンをクリックすると、入力テキストを含む X (Twitter) への投稿画面が開きます。投稿にはアプリの URL が自動的に含まれます。
